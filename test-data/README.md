## Creating Your Own Test Files

To create minimal test Exodus files for development, you can use:

1. **Python with netCDF4**: Use the `netCDF4` library to create Exodus files manually
2. **Cubit/Trelis**: Commercial meshing software with Exodus export
3. **SALOME**: Open-source meshing platform with Exodus support
4. **Gmsh + msh2exo**: Generate mesh in Gmsh and convert to Exodus

## Exodus II Format Notes

- Exodus II files are NetCDF4/HDF5 based
- Extensions: `.exo`, `.e`, `.ex2`, `.exoII`
- Contains: nodes, elements, blocks, nodesets, sidesets, variables
- Read with: `netcdf` crate in Rust, `netCDF4` in Python, SEACAS tools in C/C++
